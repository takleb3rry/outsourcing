# Cluster Agent Prompt — Comparative Evaluation Design (v2)

**Your assigned workflow cluster:** [WORKFLOW CLUSTER]

---

## Ground Rules

Before you begin, internalize these:

1. **When your confidence in a data point is low, say so.** An honest "UNSCORED — needs inquiry" is more valuable than a plausible guess. Default to UNSCORED for geographic presence and seniority density unless you can point to a specific, concrete reason for your score.
2. **Do not pad lists.** If a workflow cluster has 4 real candidates, list 4. Do not stretch to fill a quota.
3. **Score from evidence, not vibes.** If you find yourself writing "probably" or "likely" in your reasoning, that's a flag. Either find the evidence or mark it UNSCORED.

---

## Step 1: Candidate Selection

Build your list before scoring anything.

Identify **3–10** credentialing organizations whose membership would plausibly contain practitioners skilled in [WORKFLOW CLUSTER]. Include obvious candidates and less obvious ones — but stop when you run out of real candidates. Do not stretch.

For each candidate, confirm:

- It is a **credentialing organization** (exam, designation, or certification required — not a trade association or membership club)
- Its members perform work that **overlaps with this workflow cluster**
- It has a **documented, active membership base** (not aspirational or defunct)

**University Threshold:** Do not include university partners unless they are the undisputed #1 global name in a narrow specialty directly relevant to this workflow cluster (e.g., Temple for Actuarial Science, Georgia State for Risk & Insurance). "Generally prestigious" or "well-regarded program" does not meet this threshold — exclude them. If you find yourself wanting to include a university, ask: *Is there a credentialing body that serves the same population?* There almost always is. Use that instead.

Do not score yet. Produce a numbered list with a one-line rationale for each inclusion.

---

## Step 2: Score Each Organization Independently

Apply the rubric to each organization on your list. Score every organization against all three dimensions. Work through each sub-score explicitly — show your reasoning in **1–3 sentences per sub-score.** Do not look at the other organizations while doing this. Treat each one in isolation.

Produce a raw scoring table.

---

### Dimension 1: Credential Rigor & Domain Fit (50% of total)

Two sub-scores. Each scored 1–10. Average them to get the Dimension 1 score.

#### Sub-score A: Rigor (1–10)

How hard is it to earn the credential?

| Score | Description |
|-------|-------------|
| **9–10** | Multi-stage exam process (3+ exams) plus experience requirements plus continuing education. Takes 2+ years minimum to earn. |
| **7–8** | Multi-exam process or single rigorous exam plus significant experience/education requirements. |
| **5–6** | Meaningful single exam or certification process but less grueling. Months of study, not years. |
| **3–4** | Some assessment or educational requirement, but low barrier. |
| **1–2** | Essentially self-selected membership with no meaningful assessment gate. Pay-to-join. |

#### Sub-score B: Domain Fit (1–10)

How precisely does this body's membership map to this specific workflow cluster?

| Score | Description |
|-------|-------------|
| **9–10** | The credential was literally designed for people who do this exact work. Direct 1:1 match. A 10 is rare — before assigning it, ask: *Is there any meaningful filtering I'd need to do within this member pool to find people who do this work?* If yes, it's not a 10. |
| **7–8** | Strong match — the credential covers this domain as a primary focus area, though it also covers adjacent domains. |
| **5–6** | Moderate overlap — the credential covers a broader domain that includes this workflow area but also much else. Filtering within the member pool required. |
| **3–4** | Tangential — some members would have relevant skills, but the credential isn't designed for this work. |
| **1–2** | Weak relevance — membership signals general professional status but not domain-specific capability. |

**Dimension 1 Score = (Rigor + Domain Fit) ÷ 2**

That's it. Average of two 1–10 scores. No normalization.

---

### Dimension 2: Pitch Credibility (30% of total)

Score this from the **buyer's chair** — not from domain expertise. The question is: *When I name this organization in a pitch meeting with an Anthropic PM, does it help me or hurt me?*

Three sub-scores. Each scored on a **fixed scale with only the permitted values shown.** No intermediate scores are valid.

#### Gate A: Gatekeeper Credential?

| Score | Criteria |
|-------|----------|
| **4** | Yes — employers routinely require or strongly prefer this credential. Job postings list it. De facto or de jure industry standard. |
| **2** | Partially — valued and recognized but not typically required. A "nice to have." |
| **0** | No — employers don't specifically look for this credential when hiring. |

#### Gate B: Acronym Recognition?

| Score | Criteria |
|-------|----------|
| **3** | Yes — a well-read professional outside the domain has heard of this unprompted. It has crossed into general professional vocabulary (CFA, CPA, actuary). |
| **1** | Niche — respected within the domain, but a generalist needs a one-sentence explanation to place it. |
| **0** | No — requires significant explanation and the name carries no weight outside the practitioner community. |

**The 3 tier should be very small** — probably one or two organizations on your entire list, if that. Anything that requires a one-sentence explanation is a 1, not a 3. Apply this test literally: *Would a smart generalist who has never worked in this domain recognize this acronym without prompting?* If you hesitate, the answer is 1.

#### Gate C: Institutional Surface Credibility?

| Score | Criteria |
|-------|----------|
| **3** | Professional global institution — polished website, clear governance, decades of history, recognizable brand, cited by regulators. |
| **1** | Mixed — legitimate but either regional, newer, or less polished. Credible on inspection but doesn't immediately signal "major institution." |
| **0** | Weak — looks like a regional meetup, trade group, or low-budget operation from the outside. |

**Dimension 2 Score = Gate A + Gate B + Gate C**

Max = 10. Only values from the permitted set at each gate.

---

### Dimension 3: Geographic Reach & Right-Seniority Density (20% of total)

Two sub-scores, each scored 1–10. Average them to get the Dimension 3 score.

#### Sub-score A: Geographic Reach (1–10)

Score based on **meaningful presence** in each of five target recruiting regions. "Meaningful presence" means documented exam centers, local chapters, or published membership data in that region. *"Someone there could theoretically join online" does NOT count.*

**Target regions:**

1. USA & Canada
2. Latin America
3. Europe
4. Asia
5. Africa

| Regions with meaningful presence | Score |
|----------------------------------|-------|
| 5 of 5 | **10** |
| 4 of 5 | **8** |
| 3 of 5 | **6** |
| 2 of 5 | **4** |
| 1 of 5 | **2** |
| 0 of 5 | **1** |

If you cannot find documented evidence for a region, do not give credit. If you cannot confidently score any regions, mark the entire sub-score "UNSCORED — needs inquiry."

#### Sub-score B: Mid-Career Density (1–10)

What proportion of this body's active members are in the 3–8 year experience range?

| Score | Criteria |
|-------|----------|
| **9–10** | Credential is typically earned at years 3–6. Membership naturally concentrated in the target band. |
| **7–8** | Credential spans a range but has significant mid-career representation. Earned anywhere from year 2 to year 10. |
| **4–6** | Credential skews junior (earned early, many members are years 0–3) or senior (typically earned at year 10+). |
| **1–3** | Heavily skews to one end — overwhelmingly student-level or overwhelmingly executive/fellow level. |

For any score above 7: you must have a specific reason to believe mid-career practitioners are meaningfully represented — not just that the credential is theoretically available to them.

For any score below 4: confirm it genuinely skews to students or executives and isn't just an organization you have less data on. **If you're unsure, mark it UNSCORED — needs inquiry.**

**Dimension 3 Score = (Geographic Reach + Mid-Career Density) ÷ 2**

---

### Composite Score

**Total = (Dim 1 × 0.50) + (Dim 2 × 0.30) + (Dim 3 × 0.20)**

Everything is on a 1–10 scale. The composite is also 1–10. No normalization, no conversion.

---

## Step 3: Comparative Calibration Pass

Now look at all your organizations together. You are not changing scores arbitrarily — you are correcting for drift that happens when organizations are scored in isolation.

Work through each check in sequence:

**Rigor Check:** Rank your organizations by Rigor score, top to bottom. For each adjacent pair: Is the gap defensible? Is the higher-scored organization genuinely harder to earn than the one below it? If two scored the same but one is clearly harder, adjust. If a 9 and an 8 look identical on reflection, collapse them.

**Gate B Check (Recognition):** Look at all your Gate B scores together. Have you given a 3 to anything because you personally respect it, rather than because a non-specialist would recognize it unprompted? Apply the literal test again for every 3 you've assigned. Demote to 1 anything that doesn't survive.

**Domain Fit Check:** Rank your organizations by Domain Fit. The organization with the highest score should be the one whose credential was *most specifically designed* for this workflow cluster. Ask: *If I described this workflow cluster to someone and asked which credential body serves exactly this population, which one would they name first?* That organization gets the highest Domain Fit score. Everyone else is positioned relative to it.

**Seniority Check:** For any Mid-Career Density above 7, confirm you have evidence. For any below 4, confirm genuine skew vs. data gap. Convert uncertain scores to UNSCORED.

**Geographic Reach Check:** For any region you gave credit, confirm you found documented exam centers, chapters, or published membership data. Remove credit for anything that's assumption rather than evidence.

**Ordering Sanity Check:** Compute final composites. Read the ranked list. If the ordering looks wrong, **flag which specific sub-scores you believe are off and why** — but do not change them in this step. Note the concern for human review.

This last point is important: the sanity check flags problems for a human to resolve. It does not authorize you to reverse-engineer scores to match a gut ranking.

|---|---|---|---|---|---|---|---|---|---|---|---|---|
| | | | | | | | | | | | | |

Below the table:

1. **Calibration notes:** Any score that changed during Step 3 and why, in one sentence each.
2. **Sanity check flags:** If the ordering sanity check surfaced concerns, state them here with the specific sub-scores involved. Do not resolve them — flag them for human review.
3. **UNSCORED items:** List any sub-score marked "UNSCORED — needs inquiry" with the specific data point that would resolve it.
4. **Regions credited:** For each organization, list which of the 5 geographic regions you gave credit for and the basis (exam centers, chapters, membership data). This is your audit trail — if you can't fill in this section, your geographic score is wrong.
